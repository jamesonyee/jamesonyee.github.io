Overview

Designed and developed an innovative AR maze game as part of a team senior capstone project, delivering an immersive mixed-reality experience. Leveraged the advanced passthrough capabilities of the Meta Quest 3 to seamlessly blend virtual elements with the real world, enhancing user interaction and immersion. Built on Unreal Engine 5, the game features realistic physics that enable players to interact with virtual balls by tilting and manipulating a physical box, creating intuitive and engaging gameplay that bridges the gap between digital and physical environments.

Features

Key Components

1. Passthrough

The passthrough feature on the Meta Quest 3 allows users to see their real-world surroundings while interacting with virtual elements, creating a mixed-reality experience. In BallQuest AR, this allows the user to hold a real box and view the game drawn in 3D inside of it. The image above showcases the Unreal Engine 5 view of the player. The mazes are attached as static mesh components to the controller, ensuring that digital elements move in sync with the real-world box. In the final product, the controller is attached to the box, providing tactile feedback and simulating the "weight" of the virtual maze, enhancing immersion.

2. Maze Generation

3D mazes are the core of our game. Below is how we accomplished creating mazes of varying difficulty and added them to the game

We used www.mazegenerator.net/ which is a web based maze generator that generates customizable 2d mazes in various file formats for users. For this implementation we used orthonganal (square celled) mazes. The generator allows users to choose a specific height and width which is how we are able to make more difficult mazes. The generator outputs a 2d SVG which we were able to import into blender to give it a height and thickness modifier to make it a 3D game ready asset for UE5.
